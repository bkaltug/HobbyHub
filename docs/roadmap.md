The Roadmap
Phase 0 — Product definition (~1 week). Write the MVP feature list: auth, profile, unified search across 4 media types, log/rate/review, watchlist-backlog, follow users, activity feed. Everything else (comments, likes, stats pages) is v2. Sketch the ~8 core screens on paper or Figma, and draft the data model.
Phase 1 — Environment setup. Flutter SDK, Android Studio (for the Android SDK + emulator), VS Code, enable the Windows desktop target, USB debugging on your phone, Git + GitHub repo. Goal: flutter doctor fully green.
Phase 2 — API access + exploration. Get your TMDb key, Twitch/IGDB credentials, and try Open Library. Then write small Python scripts hitting each API to learn the response shapes before you write any Dart. This saves enormous time later.
Phase 3 — Backend: Supabase. For a social app you need a backend, and I'd pick Supabase over Firebase here: it's PostgreSQL, and your data is deeply relational (follows, feeds, ratings joined across media types — awkward in Firestore, natural in SQL). Set up: auth (email + Google), and tables like profiles, follows, media_items (your cache, with external_id + media_type), logs (rating, review, date, status like watched/reading/playing), lists, list_items. Add Row Level Security. Also plan to proxy the IGDB calls through a Supabase Edge Function — IGDB's OAuth tokens shouldn't live inside your app.
Phase 4 — Flutter skeleton. Project structure (feature-first folders), Riverpod for state management, go_router for navigation, theme, bottom nav (Home / Search / + Log / Profile), Supabase Flutter SDK wired in, working sign-up/sign-in screens.
Phase 5 — Search + detail pages. One search screen with tabs (Movies, TV, Books, Games), each tab backed by a repository class for its API. Media detail page with poster/cover, metadata, and action buttons. When a user interacts with an item, write it into media_items.
Phase 6 — The Letterboxd heart. Star rating (with halves), review composer, diary entries with dates, status tracking, rewatch/reread/replay counts, per-type watchlists, and the profile "favorites showcase" (Letterboxd's iconic 4 posters). This phase is what makes the app feel like Letterboxd.
Phase 7 — Social layer. Public profiles, follow/unfollow, and an activity feed built with a SQL query over your follows + logs tables. Likes/comments on reviews if time allows.
Phase 8 — Polish. cached_network_image for posters, skeleton loaders, pagination/infinite scroll, dark mode, empty states, error handling.
Phase 9 — Testing. Unit tests for your API repositories and JSON parsers, a few widget tests, plus manual testing on your Android phone and the Windows build.
Phase 10 — Release. App icon/splash, Google Play Console account ($25 one-time), a privacy policy (mandatory — you store user data), internal testing track, then production. Note TMDb's free tier is for non-commercial use — fine for a free portfolio app, but re-check terms if you ever monetize.
Phase 11 (optional, your ML playground). A small FastAPI service that does content-based recommendations ("because you rated X highly...") using genres/tags from your cached items.
